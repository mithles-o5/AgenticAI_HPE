End-to-End Request Workflow

1

LLM Tool Invocation

Claude sends a structured tool call over stdio MCP transport to mcp_server.py. Example: get_server_status("server-12")

2

Authentication Gate

MCP Server checks token_store for a valid JWT. If absent or expired, the active SSO provider (Auth0 | Okta | Local) opens a PKCE browser flow and saves the token to the store.

3

Authorization (RBAC → ABAC)

JWT claims are mapped to a role via roles.json. RBACEngine checks action permissions, then ABACEngine evaluates environment (prod restriction), time window (22:00–06:00 block), and vendor-specific rules.

4

NLP → Task Decomposition

QueryAgent.parse_query() classifies intent, action verb, and identifier. For compound instructions, TaskPlanner.decompose_instruction() splits them into dependency-ordered Task objects.

5

Resource Resolution

ResourceResolver.resolve() looks up the identifier (IP/serial/FQDN) — first in Memurai hot-cache, then PostgreSQL CMDB — to determine the owning management_source and REST routing path from routing_rules.yaml.

6

Capability Registry Lookup

AgentDispatcher queries the Capability Registry (/agents/lookup?resource_type=server&provider=oneview) to find the best OASF agent capable of handling the resolved request.

7

Task Dispatch

Execution Engine POSTs a TaskRequest payload to the resolved agent URL (e.g., http://127.0.0.1:8004/execute-task). Action verbs mapped via _ACTION_MAP to domain-specific endpoints.

8

Agent Execution → Mock Backend

Domain agent receives task, calls its mock backend REST API (e.g., HPE OneView Mock at /rest/server-hardware/{uuid}), performs the action, and returns structured results.

9

Response Synthesis

Results flow back through the MCP Server. task_planner/synthesizer.py collates multi-task results into a unified response returned to Claude as the tool result payload.
